全局设置

建议默认不保存通知正文，只保存 App ID 和分类，降低隐私风险。

阶段灯效配置

这些不是 ANCS 规则，而是设备自身状态灯效。优先级低于已匹配的 App 规则，高于完全关闭状态。

开机阶段boot
未连接时disconnected
待机standby
未匹配呼叫default.call
未匹配消息default.message

默认建议：开机短暂青绿色脉冲；未连接低亮黄灯呼吸；待机极低亮或关闭；未匹配呼叫红色闪烁；未匹配消息青绿色呼吸。

通知规则

优先级越大越先匹配。建议优先使用 App ID，其次使用 ANCS 分类。

最近收到的 App

设备收到 ANCS 通知后，会把最近 App ID 放到这里，点击即可生成规则。

提示：iOS App ID 以设备实际收到为准，不建议在固件里写死常见 App ID。

微信高级规则模板

微信的 App ID 通常可以匹配，但“语音/视频通话、@自己、红包、转账”等属于 App 内部语义，只能尝试通过标题/副标题/正文关键字识别，不能保证所有 iOS/微信版本都稳定。

稳定微信普通消息匹配 com.tencent.xin + Social，作为微信默认消息灯效。
实验微信语音/视频通话尝试用标题/正文关键字：语音通话、视频通话、来电、邀请你通话。
实验微信 @ 我尝试用关键字：@我、有人@我、提到了你。
实验微信红包尝试用关键字：红包、微信红包、给你发了一个红包。
实验微信转账尝试用关键字：转账、收款、向你转账。
建议微信兜底所有无法细分的微信通知，走统一微信灯效。

实现建议：固件先按 App ID + Category 粗匹配，再按 title/subtitle/message 关键字二次匹配；关键字规则必须允许用户自己修改。

新增/编辑规则

规则可以按 App ID、ANCS 分类、事件类型和关键字匹配。关键字匹配依赖设备是否读取标题/正文。
